00022 /*----------------------------------------------------------------------------------------------------------------------
00023 |       This is a template that can be used to create a class representing a NEXUS block. Here are the steps to follow if
00024 |       you wish to create a new block specifically for use with your particular application. Suppose your application is
00025 |       called Phylome and you want to create a private block called a PHYLOME block that can appear in NEXUS data files
00026 |       and contains commands for your program.
00027 |~
00028 |       o Copy the files nxsemptyblock.h and nxsemptyblock.cpp and rename them (e.g. nxsphylomeblock.h and 
00029 |         nxsphylomeblock.cpp)
00030 |       o In nxsphylomeblock.h and nxsphylomeblock.cpp, replace all instances of EMPTY (case-sensitive, whole word search)
00031 |         with PHYLOME
00032 |       o In nxsphylomeblock.h, replace both instances of NCL_NXSEMPTYBLOCK_H at the top of the file with
00033 |         NCL_NXSPHYLOMEBLOCK_H
00034 |       o In nxsphylomeblock.h and nxsphylomeblock.cpp, replace all instances of NxsEmptyBlock (case-sensitive, whole word
00035 |         search) with NxsPhylomeBlock
00036 |       o Modify the Read function in nxsphylomeblock.cpp to interpret what comes after the BEGIN PHYLOME command in the
00037 |         NEXUS data file
00038 |       o Modify the CharLabelToNumber and TaxonLabelToNumber if you need to read in sets of characters or taxa, 
00039 |         respectively. These functions provide a way for NxsSetReader objects to translate character or taxon labels to
00040 |         the corresponding numbers. If you do not need these capabilities, then it is safe to just delete these functions
00041 |         from nxsphylomeblock.h and nxsphylomeblock.cpp because they are no different that the base class versions
00042 |       o Modify the SkippingCommand function if you want to notify users when commands within the PHYLOME block are not 
00043 |         recognized and are being skipped
00044 |       o In nxsphylomeblock.h, replace this comment with something meaningful for your class. Start off with something
00045 |         like "This class handles reading and storage for the NEXUS block PHYLOME. It overrides the member functions 
00046 |         Read and Reset, which are abstract virtual functions in the base class NxsBlock"
00047 |~
00048 |       Adding a new data member? Don't forget to:
00049 |~
00050 |       o Describe it in the class declaration using a C-style comment. 
00051 |       o Initialize it (unless it is self-initializing) in the constructor and reinitialize it in the Reset function.
00052 |       o Describe the initial state in the constructor documentation. 
00053 |       o Delete memory allocated to it in both the destructor and Reset function. 
00054 |       o Report it in some way in the Report function. 
00055 |~
